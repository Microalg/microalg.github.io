: Calculs avec des booléens

Remarques sur Texte? et Nombre? :

  • elles se terminent par ?,
  • elles retournent des valeurs spéciales : Vrai ou Faux.

Ces valeurs spéciales Vrai et Faux sont des booléens (de George Boole). Un booléen ne peut avoir qu’une de ces deux valeurs.

Les commandes qui retournent un booléen sont appelées des prédicats. Les seuls prédicats de MicroAlg qui ne se terminent pas par ? sont :

  • =, qui teste l’égalité de deux valeurs (c’est-à-dire retourne Vrai si les deux valeurs sont égales, Faux sinon),
  • =/, qui teste la non égalité de deux valeurs,
  • concernant les nombres uniquement :
    • <, strictement inférieur
    • <=, inférieur ou égal
    • >, strictement supérieur
    • >=, supérieur ou égal

: Si

Même si ce n’est pas écrit dans le code, on peut utiliser les mots « alors » et « sinon ». Le code ci-dessus pourra se lire :

  • Afficher la question.
  • Si l’utilisateur répond « oui »,
  • Alors afficher « Tant mieux ! »,
  • Sinon afficher « Ah, qu’y a-t-il ? ».

On peut donc résumer la commande Si ainsi :
(Si booléen commande-si-vrai commande-si-faux)

Si on veut détailler, la commande Si possède trois paramètres :

  • un booléen,
  • l’instruction à exécuter si le booléen est Vrai,
  • l’instruction à exécuter si le booléen est Faux,

: Variables

Pour pouvoir utiliser une variable avec MicroAlg, il faut :

  1. tout d’abord l’initialiser, c’est-à-dire lui donner une valeur de départ du bon type, en utilisant la commande Initialiser,
  2. puis ensuite, si cette variable n’est pas une constante du programme, lui affecter une autre valeur, peut-être plusieurs fois au cours du programme.

Attention, les commandes Initialiser et Affecter_a effectuent des vérifications :

  • Initialiser
    • ne peut pas initialiser une variable qui a déjà été initialisée,
  • Affecter_a
    • ne peut pas changer la valeur d’une variable si cette variable n’a pas été initialisée auparavant,
    • ne peut pas changer le type d’une variable (la nouvelle valeur doit être du même type que l’ancienne valeur).

Pour rappel, nous connaissons pour l’instant trois types :

  • texte,
  • nombre,
  • booléen,

La nouvelle commande Type permet d’obtenir le type de certaines valeurs :

Remarque : Il n’est pas interdit d’utiliser la valeur d’une variable dans une instruction qui sera affectée à cette même variable.

Dans le premier, on accumule des nombres avec une instruction de la forme :
(Affecter_a valeur (+ valeur valeur_a_ajouter))

Ce qui signifie : « valeur prend pour nouvelle valeur le résultat du calcul valeur + valeur_a_ajouter ».

Dans le second, on accumule du texte avec une instruction de la forme :
(Affecter_a texte (Concatener texte texte_a_ajouter))

Ce qui signifie : « texte prend pour nouvelle valeur la concaténation de texte avec texte_a_ajouter ».
