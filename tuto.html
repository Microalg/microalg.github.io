Qu’est-ce ?

Cette page est un tutoriel pour MicroAlg.

Le tutoriel est assez long car il couvre tout le langage. Patience ! Peut-être faudra-t-il le faire en plusieurs fois !

Au cours de ce tutoriel, on vous proposera des petits programmes à exécuter directement dans le navigateur. N’hésitez pas à les modifier afin d’être sûre de comprendre le principe (si possible une modification rigolote).

Avant de les exécuter, essayez de deviner ce qu’ils vont faire.

: Calculs avec des booléens

Les commandes Texte? et Nombre? de la section précédente sont particulières car :

  • elles se terminent par ?,
  • elles retournent des valeurs spéciales : Vrai ou Faux.

Ces valeurs spéciales Vrai et Faux sont d’un type nouveau. Ce ne sont ni des nombres ni du texte, mais des booléens (de George Boole). Un booléen ne peut avoir qu’une de ces deux valeurs.

Les commandes qui retournent un booléen sont appelées des prédicats. Les seuls prédicats de MicroAlg qui ne se terminent pas par ? sont :

  • =, qui teste l’égalité de deux valeurs (c’est-à-dire retourne Vrai si les deux valeurs sont égales, Faux sinon),
  • =/, qui teste la non égalité de deux valeurs,
  • concernant les nombres uniquement :
    • <, strictement inférieur
    • <=, inférieur ou égal
    • >, strictement supérieur
    • >=, supérieur ou égal

: Si

Les booléens servent surtout à décider quelle instruction exécuter plutôt qu’une autre.

Par exemple, lorsque l’on discute avec quelqu’un, on peut commencer par : « Salut, ça va ? ».
Si la personne répond « oui » (qui correspondrait à Vrai), on lui répond « Tant mieux ! ».
Sinon (ce qui correspondrait à Faux), on lui répond « Ah, qu’y a-t-il ? ».

Avec MicroAlg, pour programmer ce genre de situation, on utilisera la commande Si.

Attention, dans l’exemple suivant, seul le mot « oui » en minuscule est reconnu comme une réponse positive.

Même si ce n’est pas écrit dans le code, on peut utiliser les mots « alors » et « sinon ». Le code ci-dessus pourra se lire :

  • Afficher la question.
  • Si l’utilisateur répond « oui »,
  • Alors afficher « Tant mieux ! »,
  • Sinon afficher « Ah, qu’y a-t-il ? ».

On peut donc résumer la commande Si ainsi :
(Si booléen commande-si-vrai commande-si-faux)

Si on veut détailler, la commande Si possède trois paramètres :

  • un booléen,
  • l’instruction à exécuter si le booléen est Vrai,
  • l’instruction à exécuter si le booléen est Faux,

Cette commande agit comme un aiguillage. L’analogie de l’arbre est aussi possible : l’exécution peut suivre une des deux branches, la branche du « alors » ou la branche du « sinon ».

: Variables

On a parfois besoin de stocker le résultat de (Demander) pour pouvoir se servir plusieurs fois de cette valeur. En effet, si on demande deux fois une valeur, l'utilisateur peut ne pas répondre deux fois la même chose.

Cela peut être :

  • le nom de l'utilisateur, si on veut afficher des phrases contenant son nom tout au long d'un jeu,
  • une valeur numérique qui sert plusieurs fois dans un calcul comme x dans (x+1)(x+2).

Pour pouvoir utiliser une variable avec MicroAlg, il faut :

  1. tout d’abord l’initialiser, c’est-à-dire lui donner une valeur de départ du bon type, en utilisant la commande Initialiser,
  2. puis ensuite, si cette variable n’est pas une constante du programme, lui affecter une autre valeur, peut-être plusieurs fois au cours du programme.

Attention, les commandes Initialiser et Affecter_a effectuent des vérifications :

  • Initialiser
    • ne peut pas initialiser une variable qui a déjà été initialisée,
  • Affecter_a
    • ne peut pas changer la valeur d’une variable si cette variable n’a pas été initialisée auparavant,
    • ne peut pas changer le type d’une variable (la nouvelle valeur doit être du même type que l’ancienne valeur).

Ce qui signifie : « texte prend pour nouvelle valeur la concaténation de texte avec texte_a_ajouter ».
